Notes:

When you choose a background image, be aware it may force you to alter your design. Assume by default you want to center your layout, but the image's focal point is to the left (and appears underneath your content). Your better bet is to shift the content to the left, as in this example.

Be cautious with the image you select. Don't choose something too busy. If you do choose something borderline busy, make sure it does not interfere with the text. One option is to increase the opacity of the container with your text. In this case, the background is half-transparent; the last (alpha / transparency) value in rgba could be increased.

Be sure to re-size your image so it is not massive in file size. You do not want a 1MB+ background. Be sure to adjust this in Photoshop. Remember, because it is going to be a background, it does not have to be of obscenely high quality. Keep the file size low. In this example, 'golden-gate.jpg' is 373k -- and I know I could do better.

I generally re-size the file dimensions to 2880x1920 -- the size of a 15" retina MacBook Pro. According to screensiz.es, there are only five monitors with dimensions bigger. If you try to accommodate the best monitor available, the file size of the background image is going to be ginormous ... and users with smaller screens will suffer.

In applying your background image, if you use the background-size: cover; property / value, it will scale the image as large as possible without stretching the image.

If you don't want the image to 'scroll' as content scrolls, use background-attachment: fixed;. You can see the effect here if you have a smaller screen -- the text scrolls, the background image does not move.
